安装方法

多种安装方式，选择最适合你的

1

打开 Koishi 控制台

登录你的 Koishi 实例管理后台

2

进入插件市场

点击左侧菜单的「插件市场」

3

搜索并安装

搜索 zimage 并点击安装

手动安装 
# 进入 Koishi 的 node_modules 目录
cd /path/to/koishi/node_modules

# 克隆插件
git clone https://github.com/yxl826/koishi-plugin-zimage.git

# 进入目录并安装依赖
cd koishi-plugin-zimage
npm install

# 重启 Koishi
Docker 安装 
# 复制插件到容器
docker cp koishi-plugin-zimage <容器名>:/koishi/node_modules/

# 进入容器安装依赖
docker exec -it <容器名> sh
cd /koishi/node_modules/koishi-plugin-zimage
npm install
exit

# 重启容器
docker restart <容器名>

配置说明

灵活的配置选项，满足各种需求

配置示例 
~zimage:
  apiKey: ms-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
  defaultModel: z-image-turbo
  defaultSize: 1024x1024
  defaultSteps: 8
  dailyLimit: 100
  # 违禁词配置
  bannedWords: ['敏感词1', '敏感词2']
  bannedWordsAction: reject
  msgBanned: '你的描述包含敏感内容，无法生成喵'
  # 提示语配置
  msgGenerating: '在画了喵'
  msgError: '发生错误了喵'
  msgSuccess: '喵'

必填配置

apiKey 魔搭 API Key

生成配置

defaultModel 默认: z-image-turbo
defaultSize 默认: 1024x1024
defaultSteps 默认: 8

违禁词配置

bannedWords 违禁词列表
bannedWordsAction reject/replace
msgBanned 触发违禁词提示

提示语配置

msgGenerating 在画了喵
msgError 发生错误了喵
msgSuccess 喵

模型选择

两种模型，满足不同场景

快速模型，速度优先

  • 推荐步长: 8
  • 生成速度快
  • 适合日常使用

z-image

标准模型，质量优先

  • 推荐步长: 20-50
  • 生成质量高
  • 适合精细创作

常见问题

遇到问题？这里可能有答案

提示 "Invalid model id"

确保模型名称正确，可用模型：z-image-turbo、z-image

生成超时怎么办？

可能是网络问题或服务器繁忙，可以尝试增加 maxPollTime 配置值

提示 API 错误

检查 API Key 是否正确，是否还有调用额度

如何获取魔搭 API Key？

访问 modelscope.cn 注册账号，进入"我的 AccessToken"创建 API Key

支持哪些平台？

支持所有 Koishi 支持的平台，包括 QQ、微信、Telegram、Discord 等
